# AI Airway Obstruction Detection

## Project Overview

AI Airway Obstruction Detection is a Machine Learning-based application that detects airway obstruction from voice/audio recordings. The system analyzes the input audio and predicts whether an airway obstruction is present, enabling early screening and assisting healthcare professionals.

---

##  Features

-Decoupled Architecture: Features an isolated FastAPI backend microservice alongside a high-fidelity Streamlit web client.
-Respiratory Sound Analysis: Processes `.wav` lung audio cycles using Short-Time Fourier Transform (STFT) windowing grids.
-BAAIS Core Analytics: Utilizes the Bio-Inspired Adaptive Airway Intelligence System to compute real-time structural compliance profiles.
-Longitudinal Persistence: Records transaction records inside a local SQLite ledger (`airway_history.db`) to chart multi-run risk paths.
-Automated PDF Compiler:Generates download-ready clinical diagnostic documents via automated report generation.

## 📊 Model Information & Architecture

### Ingestion & Feature Extraction Pipeline
The platform implements a dynamic Digital Signal Processing (DSP) pipeline to convert raw acoustic waves into machine-learning-ready features. Rather than analyzing flat audio data, the system applies **Short-Time Fourier Transform (STFT)** matrix windowing to break down 1D time-series breathing recordings into time-frequency spectrogram representations.

* **Acoustic Features Target:** Focuses on detecting frequency anomalies, variations in intensity, and localized wave energy patterns indicative of respiratory blockages.
* **Windowing Framework:** Processes audio sequentially using 3.0-second bounding frame matrices with a 2.5-second overlapping step configuration to prevent missing fleeting acoustic events.

### Inference & Classification Strategy
Predictions are generated by the **BAAIS (Bio-Inspired Adaptive Airway Intelligence System)** engine. This model framework computes structural compliance indexes and respiratory elasticity thresholds across the extracted acoustic frame slices.

* **Input Modality:** 1D Respiratory Breathing Sound Waveforms (`.wav` format collected via digital stethoscopes over anatomical chest landmarks).
* **Target Classes & Output Matrix:**
  * `Normal Respiratory Profile` (Healthy vesicular breath cycles)
  * `Airway Obstruction / Continuous Wheeze Profile` (High-pitched musical obstructions)
  * `Adventitious Fine/Coarse Crackles Profile` (Discontinuous explosive respiratory sounds)

---

## Dataset Specifications

The system is validated and optimized utilizing the industry-standard **ICBHI 2017 Respiratory Sound Database** (International Conference on Biomedical and Health Informatics).

* **Total Scope:** Includes 920 annotated recordings acquired from 126 unique human subjects, encompassing 6,898 distinct respiratory cycles.
* **Anatomical Mapping Protocol:** Every training file follows a strict 5-element array grid protocol parsed directly by the application pipeline:
  $$\text{[Patient ID]\_[Recording Index]\_[Chest Location]\_[Acquisition Mode]\_[Equipment Used]}$$
* **Chest Location Codes Included:** Trachea (`Tc`), Anterior Left (`Al`), Anterior Right (`Ar`), Posterior Left (`Pl`), Posterior Right (`Pr`), Lateral Left (`Ll`), Lateral Right (`Lr`).
